Guia de Integração para Desenvolvedores

Compartilhe esta página com a equipe técnica responsável pelo sistema de terceiros.

1. Autenticação

Todas as requisições para a API do GEstão AD devem incluir o cabeçalho HTTP Authorization contendo o token (API Key) gerado no padrão Bearer.

Authorization: Bearer sk_exemplo1234567890abcdef...
2. Endpoint de Teste (Validação)

Para verificar se sua chave está funcionando e se conectando corretamente à API Interna (v1).

  • URL: GET /api/v1/integration/ping
  • Retorno 200 OK: O token é válido e a API está respondendo.
  • Retorno 401 Unauthorized: O token é inválido, não foi enviado ou foi revogado.
3. Exemplo Prático de Consumo (Python)

Abaixo, um exemplo simples utilizando a biblioteca requests do Python:

import requests

url = "http://SERVIDOR_GESTAO_AD/api/v1/integration/ping"
headers = {
    "Authorization": "Bearer sk_SUA_CHAVE_AQUI"
}

response = requests.get(url, headers=headers)

if response.status_code == 200:
    print("Sucesso! Conectado à API do GEstão AD.")
    print("Resposta:", response.json())
else:
    print(f"Erro {response.status_code}: {response.json().get('error')}")
4. Exemplo Prático de Consumo (PowerShell)

Exemplo para integrações via scripts de infraestrutura em PowerShell:

$url = "http://SERVIDOR_GESTAO_AD/api/v1/integration/ping"
$headers = @{
    "Authorization" = "Bearer sk_SUA_CHAVE_AQUI"
}

$response = Invoke-RestMethod -Uri $url -Headers $headers -Method Get
Write-Output "Conectado com sucesso!"
$response
Nota sobre novos Endpoints: A fundação da API (v1) está pronta. Caso o sistema de RH precise de ações específicas (como criar ou desativar usuário enviando JSON), solicite ao administrador do GEstão AD para mapear um novo endpoint REST em routes/api.py.
